# TSP 近似算法

## 算法简介

**旅行商问题 (TSP)**：给定 $n$ 个城市和两两之间距离，求经过每个城市恰好一次并回到起点的最短回路。

TSP 是 NP-hard，以下为两种近似算法。

## 最近邻点算法 (Nearest Neighbor)

### 算法流程

1. 从任意城市出发
2. 每次选择距离当前城市**最近**的未访问城市
3. 访问所有城市后回到起点

**近似比**：$O(\log n)$，不保证最优。

### 手动模拟

距离矩阵（4 城市）：

```
    0   1   2   3
0   0  10  15  20
1  10   0  35  25
2  15  35   0  30
3  20  25  30   0
```

从 0 出发：

| 步骤 | 当前位置 | 可选 | 选择 |
|------|---------|------|------|
| 1 | 0 | 1(10), 2(15), 3(20) | **1** (最近) |
| 2 | 1 | 2(35), 3(25) | **3** (最近) |
| 3 | 3 | 2(30) | **2** |
| 4 | 2 | 回到 0(15) | **0** |

路线: 0→1→3→2→0, 总距离 = 10+25+30+15 = **80**

## 克里斯托费德斯-谢尔久科夫算法

### 算法流程

1. 求 **最小生成树** (MST)
2. 找出 MST 中**奇度顶点**（必有偶数个）
3. 对这些奇度顶点求**最小权完美匹配**
4. 合并 MST + 匹配边 → 得到欧拉图（所有顶点度数为偶）
5. 求欧拉回路
6. 沿欧拉回路**短路**（跳过重复顶点）→ 得到哈密尔顿圈

**近似比**：≤ 1.5 × OPT（需满足三角不等式）

### 关键洞察

- MST 边权和 ≤ OPT（哈密尔顿圈去掉一条边是生成树）
- 奇度顶点匹配权 ≤ 0.5 × OPT（最优 TSP 路径上交替取边）
- 总权 ≤ 1.5 × OPT

## 时间复杂度

| 算法 | 复杂度 |
|------|--------|
| 最近邻点 | $O(n^2)$ |
| Christofides | $O(n^3)$（匹配） |
