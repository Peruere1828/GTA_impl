# Hierholzer 算法：求欧拉回路/欧拉迹

## 算法简介

**欧拉回路**：经过每条边恰好一次并回到起点的回路。
**欧拉迹**：经过每条边恰好一次（不要求回到起点）的路径。

**存在条件（无向图）**：
- 欧拉回路：所有顶点度数为偶数，图连通（非零度顶点）
- 欧拉迹：恰有 0 或 2 个奇度顶点，图连通

## 算法流程

1. 找起点：若有奇度顶点，从奇度顶点出发；否则任意
2. 维护栈 `stk`，将起点入栈
3. 循环：
   - 查看栈顶顶点 $u$ 是否有未使用的邻边
   - 若有，沿该边前进到 $v$，将 $v$ 入栈，删除该边
   - 若无，将 $u$ 出栈，加入回路序列
4. 反转序列得到欧拉回路/迹

**核心思想**：不断找环，在回溯时将环拼接。

## 手动模拟示例

### 图结构（正方形+对角线）

```
0───1
|＼  |
|  ＼|
3───2
```

边: (0,1)(0,2)(0,3)(1,2)(2,3)，度: 0=3, 1=2, 2=3, 3=2

奇度顶点: 0和2，从 0 出发做欧拉迹。

### 模拟过程

| 步骤 | 栈 | 回路 | 动作 |
|------|-----|------|------|
| 始 | [0] | [] | |
| 1 | [0,1] | [] | 0→1的边 |
| 2 | [0,1,2] | [] | 1→2的边 |
| 3 | [0,1,2,0] | [] | 2→0的边 |
| 4 | [0,1,2,0,3] | [] | 0→3的边 |
| 5 | [0,1,2,0,3,2] | [] | 3→2的边 |
| 6 | [0,1,2,0,3] | [2] | 2无出边, 弹出 |
| 7 | [0,1,2,0] | [2,3] | 3无出边, 弹出 |
| 8 | [0,1,2] | [2,3,0] | 0无出边, 弹出 |
| 9 | [0,1] | [2,3,0,2] | 2无出边, 弹出 |
| 10 | [0] | [2,3,0,2,1] | 1无出边, 弹出 |
| 11 | [] | [2,3,0,2,1,0] | 0无出边, 弹出 |

**反转**：回路 = [0,1,2,0,3,2]（经过 5 条边各一次）

## 时间复杂度

- 每条边访问一次：$O(E)$
- 使用邻接矩阵查找邻边：$O(V)$ 每次，总 $O(VE)$
- 优化（邻接表+迭代器）：$O(E)$

## 测试用例

1. 正方形 C4：欧拉回路，长度=5
2. 蝴蝶图：欧拉回路，长度=7
3. 路径 0-1-2-3：欧拉迹（2 奇度点），长度=4
4. K3：欧拉回路，长度=4
5. 正方形+对角线：欧拉迹，长度=6
