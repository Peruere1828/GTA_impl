# 匈牙利算法：二分图最大匹配

## 算法简介

**问题**：给定二分图 $G=(U,V,E)$，求最大匹配——边数最多的子集，其中没有两条边共享顶点。

**匈牙利算法**：对 $U$ 中每个顶点尝试找增广路，$O(VE)$。

## 核心概念

**增广路（Augmenting Path）**：
- 从未匹配的左部顶点出发
- 交替经过非匹配边和匹配边
- 到达一个未匹配的右部顶点
- 翻转增广路上所有边的匹配状态，匹配数 +1

## 算法流程

1. 初始化所有右部顶点未匹配
2. 对左部每个顶点 $u$：
   - 从 $u$ 出发做 DFS 找增广路
   - 若找到：翻转路径，匹配数 +1
   - 若找不到：$u$ 无法匹配
3. 返回匹配数

## 手动模拟示例

### 二分图

```
左部 L={0,1,2}  右部 R={0,1,2}
边: 0-0, 0-1, 1-1, 2-1, 2-2
```

### 模拟过程

**处理 u=0**：
| 步骤 | 动作 | matchR |
|------|------|--------|
| 1 | DFS(0): 尝试 v=0, 未匹配 → 匹配 | [0,-1,-1] |

**处理 u=1**：
| 步骤 | 动作 | matchR |
|------|------|--------|
| 2 | DFS(1): 尝试 v=1, 未匹配 → 匹配 | [0,1,-1] |

**处理 u=2**：
| 步骤 | 动作 | matchR |
|------|------|--------|
| 3 | DFS(2): 尝试 v=1, 已匹配给 u=1 | |
| 4 | 递归 DFS(1): 尝试 v=1(已访问), 尝试其他邻居... 只有 v=1 | |
| 5 | u=1 无其他邻居，回溯 | |
| 6 | DFS(2): 尝试 v=2, 未匹配 → 匹配 | [0,1,2] |

**结果**：匹配数 = 3（完美匹配）
匹配对: (0,0), (1,1), (2,2)

## 时间复杂度

- 每个左部顶点做一次 DFS：$O(V)$
- 每次 DFS 访问 $O(E)$ 条边
- 总复杂度：$O(VE)$

## 测试用例

1. 简单二分图：匹配数=2
2. K2,3：匹配数=2
3. 完美匹配：匹配数=3
4. 无边：匹配数=0
5. 链式：匹配数=3
