# Kruskal 与 Prim 最小生成树算法

## 算法简介

### Kruskal 算法

**贪心策略**：按边权升序选边，并查集防环。复杂度 $O(E \log E)$。

### Prim 算法

**贪心策略**：类似 Dijkstra，从起点逐步扩展。复杂度 $O((V+E)\log V)$。

## Kruskal 流程

1. 将所有边按权值升序排序
2. 初始化并查集（每个顶点自成一个集合）
3. 按序检查每条边 $(u,v)$：
   - 若 $u,v$ 不在同一集合 → 选中，合并
   - 若在同一集合 → 跳过（成环）
4. 直到选中 $V-1$ 条边

## Prim 流程

1. 初始化 `dist[0]=0`，其他为 $\infty$
2. 重复直到所有点入 MST：
   - 选不在 MST 中 `dist` 最小的点 $u$，加入 MST
   - 对 $u$ 的每条邻边 $(u,v,w)$：若 $v$ 不在 MST 且 $w < dist[v]$，更新

## 手动模拟示例

### 图结构

边: (1,3,4), (2,0,1), (3,0,2), (4,1,2), (5,2,3), (6,0,3)

```
    0────2────1
    |\   |   /
    | \  |  /
    3───┘  /
    4
```

### Kruskal 模拟

| 步骤 | 检查边 | 权 | 集合并 | 动作 |
|------|--------|----|--------|------|
| 1 | (3,4) | 1 | {3,4} | 选 |
| 2 | (0,1) | 2 | {0,1} | 选 |
| 3 | (0,2) | 3 | {0,1,2} | 选 |
| 4 | (1,2) | 4 | 已同集 | 跳过 |
| 5 | (2,3) | 5 | {0,1,2,3,4} | 选 |

MST 总权: 1+2+3+5 = **11**

### Prim 模拟（从 0 开始）

| 步骤 | 加入 | dist 数组 | 动作 |
|------|------|-----------|------|
| 始 | - | [0,∞,∞,∞,∞] | |
| 1 | 0 | [0,2,3,6,∞] | 更新邻点 |
| 2 | 1 | [0,2,3,6,∞] | 1→2:4 不更新 |
| 3 | 2 | [0,2,3,5,∞] | 2→3:5 更新6→5 |
| 4 | 3 | [0,2,3,5,1] | 3→4:1 |
| 5 | 4 | [0,2,3,5,1] | |

MST 总权: 0+2+3+5+1 = **11** ✓

## 时间复杂度

| 算法 | 复杂度 | 适合 |
|------|--------|------|
| Kruskal | $O(E\log E)$ | 稀疏图 |
| Prim | $O((V+E)\log V)$ | 稠密图 |

## 测试用例

1. 三角形：验证两者总权一致
2. 5点标准图：Kruskal 权=11
3. 6点图：验证一致性
4. 不连通图：验证边数 < V-1
